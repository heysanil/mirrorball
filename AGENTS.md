## What it is

Mirrorball turns SSH tunnels into rows with a switch. Each enabled forward is a
supervised `ssh -N … -L/-R/-D …` process that auto-reconnects with backoff. It
drives the system `ssh` binary (no embedded SSH stack), so it inherits the user's
`~/.ssh/config`, keys, and agent. It presents as both a **menu bar** item and a
**management window**, sharing one observable state object.

## Tech stack & requirements

- **Swift 6.2**, `SWIFT_STRICT_CONCURRENCY=complete`. Keep the build warning-free
  under strict concurrency.
- **macOS 26+ deployment**, **Xcode 26+**.
- **Tuist** generates the project from `Project.swift`.
- **SwiftUI** app, `@Observable` (Observation framework — not `ObservableObject`).
- Non-sandboxed, hardened runtime on. Bundle id `co.sanil.mirrorball`.
- No third-party Swift package dependencies.

## Project layout & generation

`Project.swift` is the source of truth. `tuist generate` produces
`Mirrorball.xcodeproj` / `.xcworkspace`, both **gitignored**. Four targets:

| Target | Product | Sources | Notes |
|---|---|---|---|

## Architecture

Layered, with one-directional dependencies (`Views` → `State` → `Core`):

```
Sources/
  App/      MirrorballApp (scenes: WindowGroup + MenuBarExtra + Settings),
            AppDelegate (NSApplicationDelegateAdaptor: quit teardown, keep-alive-in-menu-bar)
  Core/     pure/foundational, no SwiftUI:
            Forward, ForwardStatus, SSHAuthMethod      — model
            SSHArguments                                — pure Forward → argv
            SupervisorConstants                         — timings
            TunnelSupervisor                            — actor: spawn/monitor/reconnect
            ChildProcessRegistry                        — kill ssh children on quit
            Persistence                                 — [Forward] ⇄ JSON
            SSHConfigParser                             — ~/.ssh/config aliases
            SecretStore (+ Keychain/InMemory), SecretUpdate
            AskpassHelper                               — writes the askpass script
            Notifier                                    — UNUserNotificationCenter
            LoginItem                                   — SMAppService.mainApp
            AppConfiguration                            — env-driven config / test seams
            DraftForward, ValidationError               — editor form state + validation
  State/    AppModel (@MainActor @Observable), ForwardEntry (@MainActor @Observable)
  Theme/    Palette (colors), Metrics (spacing + Font helpers)
  Views/    Main/ (ManagerWindow, ForwardRow, KindBadge, StatusDot, EmptyStateView, EditorTarget)
            Editor/ (ForwardEditorSheet)
            MenuBar/ (MenuBarContent, MenuBarLabel)
            Settings/ (SettingsView)
  Shared/   AccessibilityIdentifiers (A11y) — compiled into the app AND the UI test target
```

### Concurrency model (the heart of it)

```
TunnelSupervisor (actor, one per forward)
  owns the ssh Process + the reconnect state machine
        │  emits ForwardStatus over an
        ▼
  AsyncStream<ForwardStatus>            ← the Sendable bridge
        │  consumed inside a @MainActor Task
        ▼
AppModel (@MainActor @Observable)       ← single source of truth
  writes status onto the matching ForwardEntry
        │  injected via .environment(...)
        ├─────────────► ManagerWindow (WindowGroup)
        └─────────────► MenuBarContent / MenuBarLabel (MenuBarExtra)
```

- One `AppModel` is created once as `@State` in `MirrorballApp` and shared by all
  scenes through the environment. Because it's a reference type, the window and
  menu bar always reflect the same state.
- `AppModel` and `ForwardEntry` are `@MainActor @Observable` classes — **not**
  actors (SwiftUI/Observation expect main-actor access).
- The `TunnelSupervisor` is an `actor`. Its `Process` is confined to the actor;
  only `Sendable` values (`Forward`, `ForwardStatus`, strings) cross boundaries.

## The SSH engine

### Argument builder — `SSHArguments.build(for:)`

Pure function, order matters (tests assert it). After the common opts
(`-N -o ServerAliveInterval=15 -o ServerAliveCountMax=3 -o ExitOnForwardFailure=yes`):
`-p` (if `sshPort != nil/22`) → `-i` + `IdentitiesOnly=yes` (key auth) → `-J`
(jump host) → `-o <line>` per extra option → password prefs (`PreferredAuthentications`
+ `NumberOfPasswordPrompts=1`) → the `-L/-R/-D` spec → the target last. **No secret
is ever an argument.**

### Supervisor lifecycle — `TunnelSupervisor`

Constants in `SupervisorConstants`: grace `1.5s`, backoff `1s → 30s` (×2, capped),
stable-after `10s`.

1. emit `.starting`; spawn `ssh` (stdin/stdout `/dev/null`, stderr piped + drained).
2. **Grace window**: if ssh exits within 1.5s → `.error(reason)`, backoff, retry.
3. Survives grace → `.up`.
4. Monitor (poll `process.isRunning` every 100ms) until exit or stop.
5. On exit: if it had been up ≥10s, reset backoff. Then **surface the reason** —
   `exitReason(of:stderr:)` returns the last stderr line, else a non-zero exit
   code, else `nil`. Non-nil → `.error(reason)`; nil (clean exit 0) → `.reconnecting`.
   Either way, backoff + retry.
6. `stop()` cancels the supervision Task (which wakes any `Task.sleep` backoff via
   `CancellationError`), kills the child, emits `.off`, finishes the stream.

Status is **de-duplicated** (consecutive identical statuses aren't re-emitted).

### Secrets — Keychain + askpass (never argv)

- `SecretStore` protocol: `KeychainSecretStore` (generic-password items, service
  `co.sanil.mirrorball.secret`, account = forward id) in production;
  `InMemorySecretStore` in tests. Keyed by `forward.id`. Secrets are **not** on the
  `Forward` model and never persisted to JSON.
- `AskpassHelper.install(at:)` writes a tiny script that echoes
  `$MIRRORBALL_ASKPASS_SECRET`. `AppModel` installs it at launch.
- When starting a non-agent forward, `AppModel` reads the secret and passes it (+
  the askpass URL) to the supervisor, which sets `SSH_ASKPASS`,
  `SSH_ASKPASS_REQUIRE=force`, and `MIRRORBALL_ASKPASS_SECRET` **on the ssh child's
  environment only**.

### Child teardown — `ChildProcessRegistry`

Every spawned `ssh` registers here; `AppDelegate.applicationWillTerminate` calls
`terminateAll()` synchronously so quitting Mirrorball never leaks tunnels.

## State & persistence

- `AppModel` owns `[ForwardEntry]`, performs CRUD, wires supervisors, mirrors
  status, persists on every change, and routes transitions to `Notifier`.
- CRUD takes a `SecretUpdate` (`.unchanged` / `.clear` / `.set(String)`):
  `add(_:secret:)`, `update(_:with:secret:)`; `delete` also clears the secret.
- `performLaunchOnce()` is idempotent (guarded) and called from both scene
  `.task`s — requests notification auth, loads host aliases, auto-starts enabled
  forwards.
- Persistence: pretty JSON at `~/Library/Application Support/Mirrorball/forwards.json`,
  atomic writes, graceful degradation (missing/corrupt → empty list).

## UI conventions

- Native first: `List`, grouped `Form`, system `Toggle(.switch)`, SF Symbols,
  `MenuBarExtra(.window)`, `Settings` scene (gives ⌘, for free).
- The editor (`ForwardEditorSheet`) keeps the sheet open on a validation error and
  only `dismiss()`es on success. **Stored secrets are never read back into the
  form** — a "🔒 stored" state with Change/Remove is shown instead.
- Every interactive control gets an accessibility identifier from `A11y`
  (`Sources/Shared/AccessibilityIdentifiers.swift`) — the single source of truth
  shared with the UI tests.

## Configuration & test seams (env vars)

`AppConfiguration.fromEnvironment()` reads:

| Variable | Purpose |
|---|---|
| `MIRRORBALL_SSH_PATH` | substitute a fake-ssh script for `/usr/bin/ssh` |
| `MIRRORBALL_CONFIG_DIR` | redirect persistence (+ askpass script) to a temp dir |
| `MIRRORBALL_DISABLE_SIDE_EFFECTS` | `1` → skip notifications + login-item registration |
| `MIRRORBALL_SEED` | JSON array of forwards to seed on first launch |

These make the whole app deterministically testable from XCUITest and integration
tests.

## Testing

- **Unit** (`Tests/Unit`, Swift Testing): pure logic — argv, model, persistence,
  ssh-config parsing, draft validation, secret storage, notifier copy.
- **Integration** (`Tests/Integration`, Swift Testing): the supervisor and full
  `AppModel` against **real spawned processes**, driven by a fake-ssh harness
  (`SupervisorTestSupport.swift`: `makeFakeSSH(body:)`, `FakeSSH.*`,
  `collectStatuses(from:)`, `poll(timeout:_:)`). Use `InMemorySecretStore`.
- **UI** (`Tests/UI`, XCUITest): launches the app with the env seams above.
  `UITestContext` writes the fake-ssh + temp config and wires `launchEnvironment`.

Patterns:

- **TDD for bug fixes**: write a failing test that reproduces the bug first
  (e.g. a fake-ssh body that drops with a stderr line), then fix.
- The fake ssh must `exec sleep …` on "success" so the process outlives the 1.5s
  grace window.
- `xcodebuild` method-level `-only-testing` filters can resolve to 0 tests for
  Swift Testing — filter at the **target** level and read the output.

## Swift 6 concurrency rules (gotchas)

- `Process`'s `terminationHandler`/`readabilityHandler` run **off-isolation**. The
  stderr hot path uses `StderrBuffer` (an `NSLock`-guarded `@unchecked Sendable`
  class), not actor hops.
- `Process` is **not Sendable** — keep it inside the actor; pass out only results.
- `AsyncStream<ForwardStatus>` is the clean Sendable bridge to `@MainActor`.
- Pure helpers on `@MainActor` types should be `nonisolated` (e.g.
  `Notifier.message`) so non-isolated callers/tests can use them.

## Known limitations & hard-won gotchas

- **"Up" is a heuristic**, not a verified tunnel: it means "the ssh process
  survived the 1.5s grace window," not "the forward is established." A connection
  that fails *after* grace (slow auth, connect timeout) shows a brief green then
  the real error. Surfacing the exit reason mitigates this; a future improvement is
  to gate `.up` on the local listen port actually being bound for `-L`/`-D`.
- **ssh ignores private keys with loose permissions** (`0644` → "bad permissions,
  key ignored"). Keys must be `chmod 600`.
- **GUI-launched apps get a minimal environment** (no shell `PATH`, no
  `SSH_AUTH_SOCK`). Hosts whose `~/.ssh/config` uses a `ProxyCommand` or the SSH
  agent may behave differently than from a terminal.
- **XCUITest needs Accessibility/automation permission (TCC)** — Xcode prompts on
  first run; a headless/CI runner must have it pre-granted, or the runner times out
  "enabling automation mode."
- **`SMAppService.mainApp` (login-at-login)** registers reliably only for a signed
  app in a stable location (ideally `/Applications`); from DerivedData/ad-hoc it may
  throw (the Settings UI surfaces the error rather than crashing).
- Don't commit the generated `.xcodeproj`/`.xcworkspace`.

## Recipe: add a new SSH option

1. Add a defaulted, tolerant-decoded field to `Forward` (`decodeIfPresent`).
2. Emit it in `SSHArguments.build(for:)` (keep the function pure; add a unit test
   for the new flag + ordering).
3. Add the field + validation to `DraftForward.validate()` (+ a unit test).
4. Add the control to `ForwardEditorSheet` with an `A11y` identifier.
5. If it affects the live connection, confirm it flows through `TunnelSupervisor`
   (it already calls `SSHArguments.build`); add an integration test with a fake ssh
   if behavior changes.

Keep argv pure and secret-free; keep secrets in the Keychain via `SecretStore`.
